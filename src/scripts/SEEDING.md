# Database Seeding Guide

This project includes a built-in seed script that rapidly populates the database with realistic Mock data for testing and development. 

All mock data generated by this script uses a specific identifier (`seed_user_`) as a prefix to their internal IDs. This allows us to easily isolate, manage, and completely wipe generated test data without ever affecting real users and their data.

---

## Adding Seed Data

Running the seed command resolves the following test data across your database:
- **120 Mock Users** with generated realistic Arabic names and avatars.
- Realistic **Habit Categories**, **Habit Entries**, **Sync Data**, **Habit Templates**.
- Several generated **Groups**.

You have two methods to populate the database:

### Method 1: API Endpoint (Recommended)
Make a simple `GET` request to the seed endpoint. You can easily do this in your browser or Postman.

```bash
curl http://localhost:5000/api/seed
```

### Method 2: Command Line CLI
If you prefer terminal workflow, run the script directly using `ts-node`:

```bash
npx ts-node src/scripts/seed.ts
```

*(You can also map this to an `npm run seed` script inside your backend `package.json` for convenience).*

---

## Deleting Seed Data (Reverse Seed)

Deleting seed data is completely safe. It will **only** wipe records and documents that match the `seed_user_` prefix. Real user data will not be impacted, and seed members are cleanly scrubbed from any regular user groups they might have joined.

### Method 1: API Endpoint
Make a `DELETE` request to the same seed API.

Using PowerShell:
```powershell
Invoke-RestMethod -Method Delete -Uri http://localhost:5000/api/seed
```

Using Bash:
```bash
curl -X DELETE http://localhost:5000/api/seed
```

### Method 2: Command Line CLI
Add the `--reverse` flag parameter to the CLI command to trigger the reverse cleanup logic:

```bash
npx ts-node src/scripts/seed.ts --reverse
```
